# 自律狗 MVP 低保真线框

来源：[核心用户流程](./mvp-core-flow.md)

## 线框目标

这份文档定义自律狗 MVP 第一版每个核心界面上有什么、信息怎么排、用户下一步点哪里。它不是视觉稿，不定义最终颜色、插画和动效；它的作用是让产品、设计和开发对第一版 App 的屏幕结构达成一致。

核心闭环：

领养狗狗 → 创建目标 → 首页 → 完成打卡 → 狗狗反馈 → 进度页 → 节奏恢复。

## 全局导航假设

MVP 首版采用轻量导航：

- 新用户首次进入：领养页 → 目标创建页 → 首页。
- 老用户打开：直接进入首页。
- 首页底部或顶部提供进度入口。
- 目标编辑、登录保存提示、恢复任务作为局部模块或弹层出现。
- 首版不做复杂 Tab 架构，避免分散核心打卡闭环。

## 1. 领养页

页面目标：让用户按个人喜好选择一只自律狗，建立第一层情感关系。

信息层级：

1. 标题和一句短说明。
2. 狗狗列表。
3. 当前选中狗狗的性格和反馈示例。
4. 主按钮。
5. 登录保存提示。

文字线框：

```text
┌──────────────────────────────┐
│ 选一只陪你自律的小狗          │
│ 先选喜欢的，不用管目标类型    │
│                              │
│ ┌────────┐ ┌────────┐        │
│ │ 柴犬   │ │ 金毛   │        │
│ │ 热血   │ │ 温暖   │        │
│ └────────┘ └────────┘        │
│ ┌────────┐ ┌────────┐        │
│ │ 边牧   │ │ 中华田园犬 │     │
│ │ 聪明   │ │ 踏实   │        │
│ └────────┘ └────────┘        │
│                              │
│ 预览                         │
│ 慢一点也没事，我陪你把今天接回来 │
│                              │
│ [ 领养它 ]                   │
│ 登录后保存进度               │
└──────────────────────────────┘
```

关键交互：

- 点击狗狗卡片后，更新预览文案和选中态。
- 点击“领养它”后进入目标创建页。
- 用户不需要登录即可继续。

设计注意：

- 不展示“适合健身/适合学习”等绑定信息。
- 狗狗卡片重点展示品种、性格和一条反馈示例。
- 选中态要明显，但不要显得像能力选择或职业选择。

## 2. 目标创建页

页面目标：让用户在 90 秒以内创建第一个自律目标。

信息层级：

1. 和已领养狗狗的关系提示。
2. 场景选择。
3. 模板选择。
4. 目标配置。
5. 主按钮。

文字线框：

```text
┌──────────────────────────────┐
│ 先定一个今天能开始的目标      │
│ 你的狗狗会陪你完成它          │
│                              │
│ 场景                         │
│ [健身] [学习] [作息]          │
│                              │
│ 推荐模板                     │
│ ┌──────────────────────────┐ │
│ │ 运动 20 分钟              │ │
│ └──────────────────────────┘ │
│ ┌──────────────────────────┐ │
│ │ 拉伸 10 分钟              │ │
│ └──────────────────────────┘ │
│                              │
│ 目标名称                     │
│ [运动 20 分钟              ] │
│                              │
│ 频率                         │
│ [每天] [工作日] [自定义]      │
│                              │
│ 提醒时间                     │
│ [20:30]                      │
│                              │
│ [ 开始今天的节奏 ]            │
└──────────────────────────────┘
```

关键交互：

- 默认选择健身场景。
- 用户切换场景后，模板列表变化。
- 点击模板后填入目标名称。
- 点击主按钮后进入首页。

首版约束：

- MVP 最多允许 3 个目标。
- 第一次体验只引导创建 1 个目标。
- 提醒时间可以先作为配置项，不阻塞核心流程。

## 3. 首页：待完成状态

页面目标：让用户每天打开后立刻知道狗狗状态、今天要做什么、怎么完成。

信息层级：

1. 狗狗小世界。
2. 狗狗一句话。
3. 今日主目标。
4. 主打卡按钮。
5. 本周节奏简览。
6. 进度入口。

文字线框：

```text
┌──────────────────────────────┐
│ 早上好 / 晚上好              │
│                              │
│        [ 狗狗小世界 ]         │
│  小院子 / 狗窝 / 当前道具      │
│                              │
│ 今天做完 20 分钟运动，        │
│ 我就能精神满满。              │
│                              │
│ 今日目标                     │
│ ┌──────────────────────────┐ │
│ │ 运动 20 分钟              │ │
│ │ 每天 · 今天未完成          │ │
│ └──────────────────────────┘ │
│                              │
│ [ 完成今天 ]                 │
│                              │
│ 本周节奏                     │
│ ○ ● ● ○ ○ ○ ○               │
│                              │
│ 亲密度 Lv.1  12/20           │
│ [ 查看进度 ]                 │
└──────────────────────────────┘
```

关键交互：

- 点击“完成今天”进入打卡确认或直接完成打卡。
- 点击“查看进度”进入进度页。
- 若用户是游客，可以在底部轻提示“登录后保存进度”。

设计注意：

- 首页只突出一个主目标。
- 不把心情、饱腹、清洁、精力全部挤到首屏。
- 狗狗小世界和今日目标是主角，数据是辅助。

## 4. 打卡确认

页面目标：避免误触，同时让用户可以轻量补充备注和心情。

信息层级：

1. 目标确认。
2. 可选心情。
3. 可选备注。
4. 完成按钮。

文字线框：

```text
┌──────────────────────────────┐
│ 完成今天了吗？                │
│                              │
│ 今日目标                     │
│ 运动 20 分钟                  │
│                              │
│ 今天感觉                     │
│ [轻松] [还行] [有点累]        │
│                              │
│ 备注                         │
│ [今天下班后去走了两圈       ] │
│                              │
│ [ 确认完成 ]                 │
│ [ 先不填，直接完成 ]          │
└──────────────────────────────┘
```

关键交互：

- 心情和备注都是可选。
- 确认后进入狗狗反馈页。
- 同一目标同一归属日期重复完成时，不重复发放奖励。

简化方案：

- 第一版也可以不做单独确认页，在首页点击“完成今天”后直接进入反馈页。
- 若采用直接打卡，备注和心情可以放到反馈页之后的次级入口。

## 5. 狗狗反馈页

页面目标：完成后立刻给用户情绪反馈和成长反馈。

信息层级：

1. 狗狗表情或动作。
2. 一句反馈文案。
3. 状态变化。
4. 下一步按钮。

文字线框：

```text
┌──────────────────────────────┐
│        [ 开心的狗狗 ]          │
│                              │
│ 今天的节奏接住了，我们都很棒。 │
│                              │
│ 本次变化                     │
│ 亲密度 +3                    │
│ 精力 +20                     │
│                              │
│ 连续节奏                     │
│ 已保持 2 天                  │
│                              │
│ [ 看今天的进度 ]              │
│ [ 回首页 ]                    │
│                              │
│ 登录后保存这只狗狗和今天的进度 │
└──────────────────────────────┘
```

关键交互：

- 点击“看今天的进度”进入进度页。
- 点击“回首页”回到已完成状态首页。
- 游客首次打卡后展示登录保存提示，但不强制登录。

反馈规则：

- 文案由狗狗性格、目标状态、节奏状态共同决定。
- 狗狗品种不决定任务类型。
- 完成主目标至少展示一个正向变化。

## 6. 首页：已完成状态

页面目标：让用户知道今天不用重复打卡，同时鼓励明天回来。

文字线框：

```text
┌──────────────────────────────┐
│ 今天完成了                    │
│                              │
│        [ 放松的狗狗 ]          │
│                              │
│ 今天的节奏已经接住，          │
│ 明天我们继续轻轻往前。        │
│                              │
│ 今日目标                     │
│ ┌──────────────────────────┐ │
│ │ 运动 20 分钟              │ │
│ │ 已完成                    │ │
│ └──────────────────────────┘ │
│                              │
│ [ 今天已完成 ]                │
│                              │
│ 本周节奏                     │
│ ○ ● ● ○ ○ ○ ○               │
│                              │
│ [ 查看进度 ]                 │
└──────────────────────────────┘
```

关键交互：

- “今天已完成”不可重复触发奖励。
- 可允许用户查看备注或修改备注，但不影响奖励。

## 7. 进度页

页面目标：让用户看到自己和狗狗的长期变化，但不做复杂数据看板。

信息层级：

1. 亲密度和等级。
2. 本周完成率。
3. 近 7 天节奏。
4. 狗狗生活状态。
5. 回首页。

文字线框：

```text
┌──────────────────────────────┐
│ 我和狗狗的进度                │
│                              │
│ Lv.1                         │
│ 亲密度 12 / 20               │
│ [████████░░]                 │
│                              │
│ 本周完成率                   │
│ 2 / 7 天                     │
│                              │
│ 近 7 天                      │
│ 一  二  三  四  五  六  日    │
│ ○  ●  ●  ○  ○  ○  ○          │
│                              │
│ 狗狗状态                     │
│ 心情 开心                    │
│ 饱腹 75                      │
│ 清洁 65                      │
│ 精力 80                      │
│                              │
│ [ 回首页 ]                   │
└──────────────────────────────┘
```

关键交互：

- 点击近 7 天某一天可在后续版本查看备注，MVP 可不做。
- 状态值只展示，不要求用户手动管理。
- 亲密度和等级只升不降。

## 8. 首页：节奏恢复状态

页面目标：用户漏掉一天后，用更小任务把用户拉回来。

触发条件：

- 昨天应完成但没有完成。

文字线框：

```text
┌──────────────────────────────┐
│ 节奏有点乱，没关系            │
│                              │
│        [ 等你的狗狗 ]          │
│                              │
│ 昨天没接住也不代表前面白做了。 │
│ 今天做个小恢复就好。          │
│                              │
│ 今日恢复任务                 │
│ ┌──────────────────────────┐ │
│ │ 拉伸 3 分钟               │ │
│ │ 比原目标更轻一点          │ │
│ └──────────────────────────┘ │
│                              │
│ [ 做个小恢复 ]                │
│ [ 调小目标 ]                  │
│                              │
│ 亲密度不会下降                │
└──────────────────────────────┘
```

关键交互：

- 点击“做个小恢复”后进入恢复任务完成反馈。
- 点击“调小目标”进入目标编辑或目标降级模块。
- 不使用失败、断签清零、惩罚等表达。

## 9. 恢复完成反馈

页面目标：强调“回来”本身有价值。

文字线框：

```text
┌──────────────────────────────┐
│        [ 重新振作的狗狗 ]      │
│                              │
│ 回来这件事，本身就很重要。    │
│                              │
│ 本次变化                     │
│ 亲密度 +2                    │
│ 饱腹 +10                     │
│ 清洁 +10                     │
│ 精力 +10                     │
│                              │
│ 节奏状态                     │
│ 恢复中                       │
│                              │
│ [ 回首页 ]                   │
└──────────────────────────────┘
```

关键交互：

- 完成恢复任务后，节奏状态进入 recovering。
- 后续连续 2 天完成主目标后，回到 stable。

## 10. 登录保存提示

页面目标：让用户保存进度，但不打断首次体验。

出现时机：

- 首次打卡反馈页底部。
- 用户查看完整进度时。
- 用户第二次打开 App 时。
- 用户创建第二个目标时。

文字线框：

```text
┌──────────────────────────────┐
│ 保存你的自律狗                │
│                              │
│ 你已经开始了，登录后明天它还会 │
│ 在这里等你。                  │
│                              │
│ 将保存：                     │
│ 狗狗、目标、亲密度、等级、打卡 │
│                              │
│ [ 登录并保存 ]                │
│ [ 继续体验 ]                  │
└──────────────────────────────┘
```

关键交互：

- “继续体验”必须可见。
- 登录成功后迁移游客数据。
- 登录失败时不清空本地体验数据。

## 11. 长期中断降级模块

页面目标：连续 3 天以上未完成时，帮助用户降低目标难度，而不是责备用户。

文字线框：

```text
┌──────────────────────────────┐
│ 最近目标可能有点重            │
│                              │
│ 先把它调小一点，更容易回来。  │
│                              │
│ 当前目标                     │
│ 运动 20 分钟                  │
│                              │
│ 建议调整为                   │
│ [运动 10 分钟]                │
│ [拉伸 3 分钟]                 │
│ [自己修改]                    │
│                              │
│ [ 使用这个小目标 ]            │
│ [ 先保持原目标 ]              │
└──────────────────────────────┘
```

关键交互：

- 降级目标不影响历史记录。
- 亲密度和等级保留。
- 目标修改从下一次任务开始生效。

## MVP 屏幕优先级

P0 必须画清楚：

- 领养页。
- 目标创建页。
- 首页待完成状态。
- 狗狗反馈页。
- 进度页。
- 节奏恢复状态。
- 登录保存提示。

P1 可以后补：

- 打卡确认页。
- 首页已完成状态。
- 恢复完成反馈。
- 长期中断降级模块。

## 待确认问题

- 首页点击“完成今天”是直接完成，还是先进入打卡确认页？
- 进度入口放在首页底部还是右上角？
- 游客首次打卡后，登录提示用底部提示、弹层，还是反馈页内模块？
- 首屏狗狗状态是否只展示心情和亲密度？
- 长期中断降级模块是否进入 MVP 第一版，还是作为 P1？
