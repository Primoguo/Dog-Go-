## 核心方向

自律狗 iOS 版本采用温暖像素风。

这不是硬核复古游戏界面，而是一个像素化的陪伴生活空间。参考方向接近《潜水员戴夫》那种精致像素感：像素颗粒清楚，但画面有现代光影、丰富细节和舒服的生活气。

用户打开 App 时，第一眼看到的是自己的狗狗生活在院子里：有狗窝、有活动区、有目标相关道具，也有可以慢慢扩展的空间。

关键词：

- 温暖。
- 可爱但不幼稚。
- 精致像素，不是粗糙 8-bit。
- 现代光影和丰富生活细节。
- 陪伴感强。
- 可长期扩展。

参考取法：

- 学习精致像素质感、细节密度和温暖光影。
- 不照搬任何角色、场景、UI 或具体美术资产。
- 不做街机风、霓虹风或低成本像素图标风。

## 狗狗小世界定义

狗狗小世界不是一张宠物头像图，而是狗狗的生活空间。

MVP 基础场景是鸟瞰平面院子。这个院子可以理解为人类世界里别墅的院子：不是巨大的开放世界，也不是狭小宠物笼，而是一个比例协调、有生活气息、能容纳狗狗活动的空间。

视觉视角：

- 狗狗小世界是平面鸟瞰，不是站在院子里看进去的透视场景。
- 用户看到的是一张可互动的小地图：草地、石板、狗窝、入口、道具和狗狗都落在同一个平面上。
- 场景可以有轻微等距感，但不能变成强透视房间。
- 狗狗、狗窝、道具都要按地图比例摆放，像真实生活空间的平面俯视。

硬规则：

- 狗狗必须生活在场景里，而不是贴在场景上的大头像。
- 院子大小和狗狗大小要协调。
- 狗狗不能大到压住院子。
- 院子不能小到像只有一个狗窝和一只狗。
- 后期出现多只狗狗时，画面仍然要合理，不显得拥挤或夸张。

## 场景比例规则

### 单只狗 MVP

在首页狗狗小世界区域中：

| 元素 | 建议比例 |
| --- | --- |
| 狗狗高度 | 场景高度的 14%-20% |
| 狗狗宽度 | 场景宽度的 9%-15% |
| 狗窝高度 | 场景高度的 16%-24% |
| 活动空地 | 场景宽度的 58%-72% |
| 道具区 | 场景宽度的 12%-20% |
| 对话气泡 | 场景宽度的 52%-70% |

重点：

- 狗狗是视觉主角，但不是空间本身。
- 第一眼应先感受到院子地图，再发现狗狗。
- 院子要有留白，让用户感觉狗狗可以走动、趴下、运动、学习和等待。
- 狗狗脚下必须有地面、阴影或接触点，不能漂浮。

### 多只狗预留

从第一版开始，场景构图要为 2-3 只狗预留空间。

预留规则：

- 主狗位：中下方偏左或偏中。
- 第二狗位：中下方偏右，离主狗至少一个狗身宽。
- 第三狗位：后方或狗窝附近，视觉权重更低。
- 道具区不占满中央活动区。
- 对话气泡默认只绑定当前互动狗狗。

MVP 暂不显示多只狗，但构图不能把未来空间堵死。

## 像素风层级

狗狗小世界由 5 层组成：

| 层级 | 内容 | 像素风要求 |
| --- | --- | --- |
| 地图底层 | 草地、石板、土地、院子边界 | 平面鸟瞰，边缘清晰 |
| 场景层 | 围栏、狗窝、小门、树影、台阶 | 放在地图上，不做强透视远景 |
| 道具层 | 运动垫、球、水碗、书、台灯、毯子 | 小而明确，能一眼识别 |
| 狗狗层 | 当前品种和姿态 | 轮廓清楚，表情可读 |
| UI 层 | 对话气泡、狗狗状态栏、状态提示 | 像素边框，但文字保持清晰 |

## 院子构图

推荐构图：

```text
┌──────────────────────────────┐
│  [小门]      石板路            │
│      ┌──────────────┐         │
│      │   活动草地     │ [狗窝] │
│      │               │        │
│ [道具区]     [狗狗]            │
│                              │
│  栅栏 / 花圃 / 地砖边界         │
└──────────────────────────────┘
```

构图原则：

- 院子要像一张平面小地图，而不是背景插画。
- 狗窝、小门、围栏是地图元素，不能抢主狗。
- 活动空地要明显，给狗狗移动和未来多狗互动留空间。
- 道具靠近狗狗，但不能比狗狗更大。
- 院子边界要明确，让用户知道这是狗狗的家。
- 可以用石板路、草地边界、栅栏和花圃建立空间秩序。

禁止：

- 使用强透视天空、远景和大房子压住平面感。
- 把狗狗画成正面大头像放在院子上。
- 让狗窝、别墅或背景建筑成为画面主角。

## 狗狗像素规格

首发狗狗：

- 柴犬。
- 金毛。
- 边牧。
- 中华田园犬。

像素表现要求：

- 每只狗用不同轮廓和配色区分，而不是只换颜色。
- 表情重点放在眼睛、耳朵、尾巴和身体姿态。
- 不做过度写实毛发。
- 不做超大头小身的夸张比例。
- 狗狗体型要适合院子比例。
- 小世界里的狗狗是平面地图比例的小狗。
- 点击狗狗后，状态栏中可以显示更清楚的狗狗半身或站姿像素形象。

## 独特外观生成

用户选择的是品种，但每条狗狗的具体外观是独一无二的。

生成规则：

- 品种决定基础轮廓：耳朵、尾巴、体型、脸型。
- `appearanceSeed` 决定稳定随机结果。
- 配色组合从该品种允许的色板中随机抽取。
- 同一只狗每次打开 App 外观保持一致。
- 不同用户或未来多只狗即使同品种，也可以有不同花色。

可随机项：

| 项目 | 示例 |
| --- | --- |
| 主毛色 | 赤棕、奶油、黑白、黄褐、浅棕 |
| 次毛色 | 胸口、耳尖、尾巴尖、脚掌 |
| 斑纹 | 额头小斑、背部色块、耳朵深色 |
| 配饰 | 小项圈、轻量小牌子 |
| 表情细节 | 眼睛亮点、眉点、嘴角 |

限制：

- 随机不能破坏品种识别。
- 配色不能过于花哨。
- 配饰不能像商业皮肤系统一样抢戏。
- MVP 可先实现主毛色、次毛色、项圈色三项。

## 狗狗状态栏

点击狗狗后，不在小世界里放大狗狗，也不弹出突兀弹窗，而是在狗狗小世界内侧底部出现一个狗狗状态栏。

推荐位置：

- 放在狗狗小世界底部内侧。
- 高度约占狗狗小世界高度的 18%-24%。
- 默认收起或半透明轻显示。
- 点击狗狗后展开。
- 不遮挡今日目标和主按钮。
- 不遮挡狗狗在地图上的位置。

状态栏结构：

```text
┌──────────────────────────────┐
│ [狗狗像素头像]  阿土 · 中华田园犬 │
│ Lv.1  期待中    亲密度 12/20    │
└──────────────────────────────┘
```

展示内容优先级：

1. 狗狗更清楚的像素形象。
2. 狗狗名字和品种。
3. 当前状态：期待中、开心、等待、恢复中。
4. 亲密度或等级。
5. 一句短对话。

交互规则：

- 点击狗狗：展开状态栏并显示狗狗清晰形象。
- 点击状态栏外的小世界空白：状态栏收起。
- 完成任务：状态栏可短暂显示奖励变化。
- 多狗版本：点击哪只狗，状态栏展示哪只狗。

视觉规则：

- 状态栏像院子里的信息托盘，不像系统弹窗。
- 使用像素边框和浅色底。
- 背景要有透明度或柔和色，不压住小世界。
- 状态栏里的狗狗图可以比地图里的狗狗更大，但仍然是信息展示，不替代地图狗狗。

基础姿态：

| 姿态 | 用途 | 表现 |
| --- | --- | --- |
| idle | 待完成 | 站着或坐着看向用户 |
| focused | 学习、运动准备、点击互动 | 稍微前倾或注意力集中 |
| happy | 完成任务 | 摇尾巴、轻跳、靠近 |
| waiting | 漏一天、恢复任务 | 靠近门口或狗窝，不责备 |
| rest | 作息目标 | 趴窝、打哈欠、放松 |

## 像素 UI 方向

App 界面整体也采用像素风，但要保留 iOS App 的可用性。

UI 规则：

- 按钮可有像素边框或硬朗边缘，但点击区域必须足够大。
- 卡片可以像像素面板，但不做复杂游戏 HUD。
- 文字使用系统字体优先，必要时只在标题或装饰标签中使用像素字体。
- 主操作按钮必须清晰，不为了风格牺牲可读性。
- 圆角减少，但不完全变成尖锐方块。

建议：

- 大卡片圆角：6px-8px。
- 小标签圆角：4px-6px。
- 像素边框：1-2px。
- 阴影：少量块状阴影，不做浓重发光。

## 色彩方向

整体色彩：温暖、干净、低压力。

建议色组：

| 用途 | 方向 |
| --- | --- |
| 背景 | 暖米白、浅天空蓝、浅草绿 |
| 主按钮 | 深草绿或暖棕绿 |
| 强调色 | 金黄、番茄红、湖蓝少量点缀 |
| 院子 | 草绿、石板灰、木色、暖墙色 |
| 恢复状态 | 不用灰败，使用安静的低饱和蓝绿或暖灰 |

禁止：

- 全屏黑暗复古游戏感。
- 大面积霓虹色。
- 过度单一绿色或棕色。
- 断签后把场景做破败、枯萎、脏乱。

## 动效方向

MVP 采用轻量像素动效。

优先级：

1. 点击狗狗：狗狗抬头或看向用户。
2. 完成任务：狗狗摇尾巴或轻跳。
3. 恢复任务：狗狗从门口或狗窝旁靠近。
4. 目标呼应：运动时小跑，学习时陪读，作息时趴窝。

动效规则：

- 动作短，0.4-1.2 秒。
- 不做持续高频闪动。
- 不抢今日目标按钮。
- 不让用户感觉必须玩游戏才能完成任务。

## 首页落地规则

首页第一屏仍然采用上下结构：

```text
顶部问候
狗狗像素小世界
今日目标和打卡按钮
轻量进度
```

调整重点：

- 狗狗小世界高度由屏幕宽度决定，避免超出手机宽度。
- 场景要按院子比例绘制，不按狗狗头像比例绘制。
- 狗狗小世界上半屏不能塞太多 UI。
- 今日目标区保持清晰和稳定。
- 调试按钮只允许 Debug 环境出现。

## MVP 资产清单

第一版最小像素资产：

| 资产 | 数量 | 说明 |
| --- | --- | --- |
| 基础像素鸟瞰院子 | 1 | 白天默认 |
| 院子恢复氛围 | 1 | 更安静，不破败 |
| 狗狗品种 | 4 | 柴犬、金毛、边牧、中华田园犬 |
| 每只狗姿态 | 3-5 | idle、happy、waiting 优先 |
| 目标道具组 | 3 | 健身、学习、作息 |
| 对话气泡样式 | 1 | 像素边框，文字清晰 |
| 狗狗状态栏 | 1 | 点击狗狗后展开 |

## 当前决策

从 2026-06-24 起，iOS MVP 的视觉方向从温暖插画风调整为温暖像素风。

旧的插画风原型和 Web MVP 只保留为流程验证参考，不作为最终视觉依据。
