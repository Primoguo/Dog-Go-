# 自律狗狗小世界概念

来源：[核心用户流程](./mvp-core-flow.md)、[低保真线框](./mvp-wireframes.md)

## 核心判断

狗狗不是自律狗 App 的装饰元素，而是产品的核心界面。

用户打开 App 时，第一眼看到的应该不是任务列表，而是自己的狗狗正在一个小世界里生活。任务、打卡、进度和恢复都围绕这个小世界发生。这样产品的情绪中心从“我要完成任务”变成“我和它一起把今天接住”。

## 产品定位

这个上半屏场景可以命名为：狗狗小世界。

狗狗小世界承载三件事：

- 陪伴感：狗狗在等你、看见你、回应你。
- 任务呼应：用户今天做什么，狗狗也进入相似氛围。
- 节奏恢复：用户中断后，狗狗仍然在，不责备，只帮助回来。

一句话定义：

狗狗小世界是自律狗的情感主界面，任务卡片只是它下面的行动入口。

## 页面结构建议

首页采用上下结构：

```text
┌──────────────────────────────┐
│                              │
│        狗狗小世界             │
│  场景 / 狗狗 / 动作 / 对话气泡 │
│                              │
├──────────────────────────────┤
│ 今日目标                     │
│ 运动 20 分钟                  │
│                              │
│ [ 完成今天 ]                 │
│ 本周节奏 / 亲密度 / 进度入口  │
└──────────────────────────────┘
```

信息优先级：

1. 狗狗当前状态和动作。
2. 狗狗对用户说的一句话。
3. 今日任务。
4. 打卡按钮。
5. 亲密度、本周节奏等辅助信息。

## 狗狗和任务的关系

狗狗品种不绑定任务。

用户按喜好选择柴犬、金毛、边牧、中华田园犬。任何狗狗都可以陪用户运动、学习、作息或恢复节奏。

但狗狗当前动作可以呼应任务。

例如：

| 用户目标 | 狗狗动作 | 场景道具 |
| --- | --- | --- |
| 运动 | 小跑、伸展、叼球、绕院子走 | 球、运动垫、水碗、毛巾 |
| 学习 | 趴在书旁、安静陪读、抬头看用户 | 书、本子、台灯、书桌 |
| 作息 | 打哈欠、趴窝、整理毯子 | 狗窝、毯子、月亮灯 |
| 恢复任务 | 在门口等、慢慢靠近、轻摇尾巴 | 门口、小脚印、小垫子 |
| 已完成 | 摇尾巴、跑过来、趴在用户旁边 | 亮一点的灯、装满的水碗 |

关键原则：

- 品种决定语气和外观。
- 目标决定动作和道具。
- 节奏状态决定情绪氛围。

## 场景设定

MVP 建议只做一个基础场景，不做多个完整场景。

推荐基础场景：小院子。

基础元素：

- 一块活动区。
- 一个狗窝。
- 一小块草地或地垫。
- 一个可替换道具区。
- 一处可以表现“等待用户回来”的入口，比如小门或台阶。

为什么选小院子：

- 比室内房间更有呼吸感。
- 狗狗运动、休息、等待、互动都自然。
- 可以用道具变化表达不同目标，不必频繁切换完整背景。
- 后续容易扩展为小屋、别墅、花园、书房角落等升级空间。

MVP 场景变化：

| 状态 | 场景变化 |
| --- | --- |
| 待完成 | 普通白天或当前时间氛围，狗狗看向用户 |
| 运动目标 | 出现球、运动垫、水碗等道具 |
| 学习目标 | 出现书、本子、台灯等道具 |
| 作息目标 | 出现毯子、窝、柔和灯光等道具 |
| 已完成 | 场景更亮，狗狗更开心，水碗或小道具状态变好 |
| 漏 1 天 | 氛围轻微安静，狗狗在门口等 |
| 恢复中 | 狗狗靠近用户，场景从安静变暖 |
| 长期中断 | 场景不破败，不惩罚，只更安静、更简化 |

## 视觉风格方向

### 方向 A：温暖插画风

推荐作为 MVP 首选。

特点：

- 成人感更强，不容易幼稚。
- 狗狗表情和动作容易传达情绪。
- 场景氛围柔软，适合陪伴产品。
- 开发成本可控，可以先用少量静态图和局部动画实现。

适合自律狗的原因：

- 产品核心是陪伴和关系，不是游戏闯关。
- 温暖插画风可以让“断了也能回来”更可信。

### 方向 B：精致像素风

特点：

- 记忆点强。
- 状态变化和道具变化容易游戏化。
- 适合做长期养成和装饰系统。

风险：

- 容易显得像小游戏。
- 如果像素精度不够，会显得廉价。
- 成人自律用户可能觉得过于娱乐化。

建议：

- 可以作为后续主题皮肤或视觉探索。
- 不建议 MVP 第一版直接押注，除非团队已经非常确定像素风审美。

### 方向 C：轻 2.5D 场景风

特点：

- 空间感强，质感高级。
- 很适合小院子、小屋、别墅等成长空间。

风险：

- 美术和动效成本更高。
- 工程实现成本更高。
- MVP 验证阶段容易拖慢速度。

建议：

- 作为 V2 或商业化空间升级方向。
- MVP 可以在构图上预留 2.5D 的空间感，但先用插画实现。

## MVP 推荐方案

第一版采用：温暖插画风 + 单一小院子 + 少量状态动作。

MVP 最小资产：

| 类型 | 数量 | 说明 |
| --- | --- | --- |
| 基础小院子背景 | 1 | 白天默认场景 |
| 狗狗品种形象 | 4 | 柴犬、金毛、边牧、中华田园犬 |
| 每只狗基础姿态 | 3 | 待完成、开心、等待 |
| 任务道具组 | 3 | 运动、学习、作息 |
| 恢复状态道具 | 1 | 门口、脚印或小垫子 |
| 对话气泡 | 1 套 | 支持不同文案 |

MVP 最小状态：

| 状态 | 狗狗表现 |
| --- | --- |
| 新用户领养 | 看向用户，轻微期待 |
| 今日待完成 | 在场景里做与目标相关的轻动作 |
| 今日已完成 | 开心、靠近、摇尾巴或趴下放松 |
| 漏 1 天 | 在门口或狗窝旁等待 |
| 恢复完成 | 重新振作，靠近用户 |

第一版可以先不做：

- 多个场景完整切换。
- 复杂连续动画。
- 家具摆放系统。
- 皮肤商城。
- 3D 空间。
- 狗狗自由移动 AI。

## 交互设计

### 点击狗狗

用户点击狗狗时：

1. 狗狗看向用户。
2. 出现一句对话。
3. 如果今天还没完成，文案轻轻提醒。
4. 如果今天已完成，文案肯定用户。
5. 如果节奏中断，文案强调可以回来。

示例：

- 待完成：今天先动一下就好，我在这儿等你。
- 已完成：今天已经接住了，休息也算节奏的一部分。
- 恢复：慢慢来，我们把门重新打开。

### 完成任务

用户完成任务后：

1. 狗狗动作变开心。
2. 对话气泡出现反馈。
3. 状态变化浮现，比如亲密度 +3、精力 +20。
4. 场景有轻微积极变化，比如灯更亮、水碗更满。

### 中断后回来

用户漏掉一天后打开 App：

1. 狗狗不责备。
2. 场景略安静。
3. 狗狗在门口或窝旁等待。
4. 今日任务变成更小的恢复任务。
5. 文案强调“回来比完美重要”。

## 后续成长空间

后续版本可以逐步扩展：

- 小院子升级为小屋、花园、别墅。
- 解锁更多家具和道具。
- 根据连续节奏让场景更丰富。
- 节奏恢复后出现“修好小路”“重新点亮灯”等轻叙事。
- 用户可以选择视觉主题，比如插画风、像素风。
- 狗狗有更多动作：跑、坐、趴、伸懒腰、叼球、看书、打哈欠。

商业化方向可以放在：

- 更多狗狗品种。
- 场景主题。
- 狗窝样式。
- 道具装饰。
- 季节场景。

但 MVP 不应把商业化放在核心闭环之前。

## 设计原则

- 狗狗永远不惩罚用户。
- 场景可以安静，但不能破败羞辱。
- 任务呼应通过动作和道具表达，不通过品种绑定表达。
- 对话要像陪伴，不像教练训话。
- 上半屏承载关系，下半屏承载行动。
- 场景变化要服务打卡闭环，不要喧宾夺主。

## 待确认问题

- MVP 视觉风格是否确定为温暖插画风？
- 基础场景是否确定为小院子？
- 第一版是否需要狗狗轻动画，还是静态姿态 + 对话气泡即可？
- 狗狗动作按目标类型变化，还是先只按节奏状态变化？
- 领养页是否也展示狗狗小世界预览？
