# 狗狗小世界 MVP 规格

来源：[狗狗小世界概念](./dog-scene-concept.md)、[低保真线框](./mvp-wireframes.md)

## 规格目标

这份文档把“狗狗小世界”从概念转成 MVP 可执行规格。设计可以据此画首页视觉稿，美术可以据此拆资产，开发可以据此搭状态和交互。

MVP 决策：

- 视觉方向：温暖像素风。
- 基础场景：单一小院子。
- 表现方式：静态姿态 + 少量轻动画 + 对话气泡。
- 场景策略：不做多场景切换，用道具和狗狗状态表达任务变化。
- 核心位置：首页上半屏。
- 比例原则：院子是狗狗的生活空间，狗狗大小必须和院子、狗窝、活动区保持协调，不能像放大的宠物头像。
- 视角原则：狗狗小世界采用鸟瞰平面视角，像一张可互动的小院地图，不做强透视室内或院内视角。

## 首页布局规格

首页采用上下结构：

```text
┌──────────────────────────────┐
│ 顶部问候 / 保存提示           │
├──────────────────────────────┤
│                              │
│        狗狗小世界             │
│    小院子 + 狗狗 + 道具 + 气泡 │
│                              │
├──────────────────────────────┤
│ 今日目标卡片                  │
│ 主打卡按钮                    │
│ 本周节奏 / 亲密度 / 进度入口  │
└──────────────────────────────┘
```

推荐比例：

| 区域 | 高度占比 | 说明 |
| --- | --- | --- |
| 顶部问候 | 8%-12% | 早上好、晚上好、登录保存提示 |
| 狗狗小世界 | 42%-50% | 首页情感主视觉 |
| 今日行动区 | 38%-46% | 目标、打卡、节奏、进度 |

设计约束：

- 狗狗小世界必须在首屏可见，不需要用户滚动。
- 今日打卡按钮必须在首屏可见。
- 狗狗对话气泡不能遮挡今日目标按钮。
- 上半屏以场景和狗狗为主，不放复杂数据。

## 狗狗小世界画面结构

MVP 小院子由 5 层组成：

| 层级 | 内容 | 是否固定 |
| --- | --- | --- |
| 地图底层 | 草地、地砖、石板路、院子边界 | 固定 |
| 场景层 | 狗窝、入口、小门、栅栏、花圃 | 固定 |
| 道具层 | 运动、学习、作息、恢复道具 | 按目标和状态切换 |
| 狗狗层 | 当前品种 + 当前姿态 | 按用户选择和状态切换 |
| UI 层 | 对话气泡、狗狗状态栏、状态变化浮层 | 按交互出现 |

像素风补充：

- 远景、院墙、狗窝、道具和狗狗都采用像素化绘制。
- UI 层可以有像素边框，但文字必须保持清晰。
- 场景要为未来多只狗预留空间，MVP 单只狗不能占满画面。
- 小世界里的狗狗按平面地图比例展示，点击后在状态栏里显示更清楚的狗狗样子和品种。

文字线框：

```text
┌──────────────────────────────┐
│ [小门]    石板路       [狗窝] │
│                              │
│ [道具区]    活动草地           │
│              [狗狗]           │
│                              │
│ [点击狗狗后展开狗狗状态栏]     │
└──────────────────────────────┘
```

## 固定场景元素

MVP 固定元素：

- 小院子背景。
- 狗窝。
- 一块活动区。
- 一处入口或小门。
- 一个可替换道具区。
- 一个可展开的狗狗状态栏。

固定元素要求：

- 场景不能显得破败、脏乱或惩罚化。
- 即使用户中断，场景也只是更安静，不做枯萎、损坏、灰暗压迫等表达。
- 狗窝和入口要给“它在等你”的叙事留空间。
- 可替换道具区要足够明确，方便切换运动、学习、作息道具。

## 可变道具规格

道具根据目标类型和节奏状态切换，但不改变狗狗品种。

| 目标或状态 | 道具 | 用途 |
| --- | --- | --- |
| 健身目标 | 球、运动垫、水碗、毛巾 | 呼应运动任务 |
| 学习目标 | 书、本子、台灯、小书桌 | 呼应学习任务 |
| 作息目标 | 毯子、月亮灯、睡觉窝 | 呼应作息任务 |
| 恢复任务 | 小门、脚印、小垫子 | 表达回来、重新开始 |
| 已完成 | 更亮的灯、装满的水碗、小星标 | 表达今天被接住 |

MVP 最小实现：

- 每类目标先做 1 组道具。
- 道具只需要出现或替换，不要求复杂动画。
- 若工程时间紧，先只做健身、学习、作息 3 组道具，恢复状态复用小门和脚印。

## 狗狗品种规格

首发品种：

- 柴犬。
- 金毛。
- 边牧。
- 中华田园犬。

选择原则：

- 用户按喜好选择狗狗。
- 品种不绑定任务类型。
- 所有品种都支持所有目标。
- 品种只影响外观、性格标签和反馈语气。
- 每条狗狗在品种基础上有稳定随机的配色组合，保证同品种狗狗也可以独一无二。
- 随机外观不能破坏品种识别，MVP 优先支持主毛色、次毛色、项圈色。

每个品种最少需要 3 个姿态：

| 姿态 | 用途 | 表情方向 |
| --- | --- | --- |
| idle | 今日待完成、普通等待 | 期待、看向用户 |
| happy | 完成打卡、连续节点 | 开心、靠近、摇尾巴 |
| waiting | 漏 1 天、恢复任务 | 安静等待、不责备 |

推荐第 4 个姿态：

| 姿态 | 用途 | 表情方向 |
| --- | --- | --- |
| focused | 学习、作息、用户点击时 | 专注、陪伴、轻轻回应 |

MVP 资产数量：

| 内容 | 最小数量 | 推荐数量 |
| --- | --- | --- |
| 狗狗品种 | 4 | 4 |
| 每只狗姿态 | 3 | 4 |
| 狗狗总姿态图 | 12 | 16 |

## 状态矩阵

首页狗狗小世界根据两个维度决定展示：

- 今日目标类型：健身、学习、作息。
- 节奏状态：待完成、已完成、恢复、长期中断。

### 待完成

| 目标类型 | 狗狗姿态 | 道具 | 对话方向 |
| --- | --- | --- | --- |
| 健身 | idle 或 focused | 球、运动垫 | 先动一下就好 |
| 学习 | focused | 书、本子 | 陪你坐一会儿 |
| 作息 | idle 或 focused | 毯子、月亮灯 | 今天慢慢收尾 |

### 已完成

| 目标类型 | 狗狗姿态 | 道具 | 对话方向 |
| --- | --- | --- | --- |
| 健身 | happy | 水碗、毛巾 | 今天身体动起来了 |
| 学习 | happy | 书、本子 | 你把注意力接住了 |
| 作息 | happy | 毯子、柔和灯 | 照顾自己也算进步 |

### 恢复任务

| 状态 | 狗狗姿态 | 道具 | 对话方向 |
| --- | --- | --- | --- |
| 漏 1 天 | waiting | 小门、脚印 | 昨天乱了也没关系 |
| 恢复中 | idle 或 happy | 小垫子、暖灯 | 回来这件事很重要 |
| 长期中断 | waiting | 简化场景 | 把目标调小一点 |

## 交互规格

### 点击狗狗

触发：用户点击狗狗。

表现：

1. 狗狗短暂切到看向用户的状态。
2. 狗狗状态栏从小世界底部内侧展开。
3. 状态栏显示狗狗清晰像素形象、名字、品种、等级和当前状态。
4. 短对话可以显示在状态栏内，减少地图遮挡。
5. 点击小世界空白处后状态栏收起。

文案来源：

- 品种语气。
- 今日目标状态。
- 节奏状态。

禁止：

- 点击狗狗后弹出复杂菜单。
- 点击狗狗后跳出页面。
- 点击狗狗后出现突兀的居中弹窗。
- 在地图里把狗狗临时放大。
- 使用责备、惩罚、羞辱文案。

### 点击完成今天

触发：用户完成主目标。

表现：

1. 狗狗切到 happy。
2. 对话气泡出现完成反馈。
3. 状态变化浮层出现：亲密度 +3、精力或其他状态变化。
4. 今日行动区切到已完成状态。

轻动画建议：

- 狗狗轻微跳动或摇尾巴。
- 状态变化数字向上浮动。
- 道具状态轻微变亮。

### 进入恢复状态

触发：用户昨天未完成目标。

表现：

1. 狗狗切到 waiting。
2. 场景氛围略安静。
3. 今日目标区显示恢复任务。
4. 对话强调“回来就好”。

禁止：

- 场景破败。
- 狗狗生气。
- 亲密度下降。
- 使用连续天数清零作为主视觉。

## 对话气泡规格

气泡位置：

- 优先在狗狗上方或侧上方。
- 不遮挡狗狗脸部。
- 不遮挡打卡按钮。
- 移动端宽度不超过场景宽度的 80%。

气泡内容：

- 1 句话为主。
- 最长不超过 24 个中文字符，特殊情况下不超过 32 个。
- 避免说教。
- 避免解释功能。

气泡状态：

| 场景 | 是否自动出现 |
| --- | --- |
| 首次进入首页 | 是 |
| 点击狗狗 | 是 |
| 完成打卡 | 是 |
| 恢复任务出现 | 是 |
| 已完成状态反复进入 | 可选 |

## 状态变化浮层

用途：让用户看到打卡带来的成长反馈。

内容：

- 亲密度 +3。
- 精力 +20 或其他状态。
- 连续 3 天、7 天时展示额外奖励。

表现：

```text
┌──────────────────────────────┐
│        [ 开心狗狗 ]            │
│                              │
│      亲密度 +3                │
│      精力 +20                 │
└──────────────────────────────┘
```

约束：

- 浮层不超过 2-3 条变化。
- 不做复杂结算页。
- 不出现负向扣分作为主反馈。

## MVP 不做

第一版明确不做：

- 多个完整场景。
- 3D 狗狗。
- 狗狗自由移动 AI。
- 家具摆放系统。
- 场景商城。
- 复杂连续动画。
- 根据品种限制任务。
- 用户中断后的破败、脏乱、惩罚视觉。

## 验收标准

设计验收：

- 首页第一眼能看出狗狗在一个小世界里，而不是孤立头像。
- 用户能理解狗狗当前状态：等你、开心、恢复中。
- 运动、学习、作息至少能通过道具或动作区分。
- 狗狗对话像陪伴，不像系统提示。
- 狗狗品种不暗示任务绑定。

开发验收：

- 首页能根据品种切换狗狗资产。
- 首页能根据目标类型切换道具组。
- 首页能根据节奏状态切换狗狗姿态。
- 点击狗狗能触发对话气泡。
- 完成打卡后能切换 happy 状态并展示状态变化。
- 恢复任务出现时能切换 waiting 状态和恢复道具。

产品验收：

- 用户进入首页先注意到狗狗，再注意到任务。
- 用户完成任务后能感到狗狗在回应自己。
- 用户中断后不会感到被责备。
- 狗狗小世界服务核心打卡闭环，不抢走主任务。
