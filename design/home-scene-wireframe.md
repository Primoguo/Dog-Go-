# 自律狗首页视觉线框

来源：[狗狗小世界 MVP 规格](./dog-scene-mvp-spec.md)、[狗狗反馈文案库](./dog-feedback-copy.md)、[MVP 低保真线框](./mvp-wireframes.md)

## 目标

这份文档细化首页第一屏的视觉结构，把“狗狗小世界”和“今日目标”放到同一个可设计、可开发的首页线框里。

首页要达成三件事：

- 用户第一眼看到狗狗在小世界里生活。
- 用户第二眼知道今天要做什么。
- 用户不思考就能完成打卡。

## 设计方向

MVP 首页采用：

- 温暖像素风。
- 单一小院子场景。
- 上半屏承载关系。
- 下半屏承载行动。
- 狗狗品种按用户喜好选择，不绑定任务。
- 狗狗动作和道具根据目标类型与节奏状态变化。

## 首页总结构

```text
┌──────────────────────────────┐
│ 顶部问候区                    │
│ 早上好，今天也慢慢来          │
├──────────────────────────────┤
│                              │
│        狗狗小世界             │
│  小院子 / 狗狗 / 道具 / 气泡   │
│                              │
├──────────────────────────────┤
│ 今日行动区                    │
│ 今日目标卡片                  │
│ 主按钮                        │
│ 本周节奏 / 亲密度 / 进度入口  │
└──────────────────────────────┘
```

推荐比例：

| 区域 | 高度占比 | 设计重点 |
| --- | --- | --- |
| 顶部问候区 | 8%-10% | 轻，不抢狗狗 |
| 狗狗小世界 | 44%-48% | 首页情感主视觉 |
| 今日行动区 | 42%-46% | 目标和打卡效率 |

## 1. 顶部问候区

目标：给用户一个柔和进入感，不做复杂导航。

内容：

- 时间问候：早上好、下午好、晚上好。
- 简短陪伴文案。
- 可选登录保存提示。
- 可选进度入口或设置入口。

线框：

```text
┌──────────────────────────────┐
│ 早上好                       │
│ 今天也慢慢来                 │
│                    [进度]    │
└──────────────────────────────┘
```

规则：

- 顶部文案不超过 2 行。
- 不放复杂统计。
- 游客状态可显示“登录后保存进度”，但不要做成强弹窗。

## 2. 狗狗小世界构图

目标：让狗狗看起来生活在一个真实的平面小院地图里，而不是孤立贴图。

视角：

- 狗狗小世界采用鸟瞰平面视角。
- 它像一张可互动的小院地图，不是站在院子里向内看的透视画。
- 狗狗、狗窝、道具都按地图比例摆放。
- 点击狗狗后，才在狗狗状态栏里显示更清楚的狗狗样子和品种。

### 基础构图

```text
┌──────────────────────────────┐
│ [小门]     石板路       [狗窝] │
│                              │
│       ┌────────────┐         │
│ [道具]│   活动草地   │         │
│       │    [狗狗]   │         │
│       └────────────┘         │
│ 花圃 / 栅栏 / 地砖边界         │
│ [点击后：狗狗状态栏从底部展开] │
└──────────────────────────────┘
```

布局建议：

| 元素 | 位置 | 说明 |
| --- | --- | --- |
| 狗狗主体 | 活动草地中下方 | 地图中的当前狗狗 |
| 狗窝 | 右上或右侧 | 固定生活点 |
| 道具区 | 左侧或草地边缘 | 按目标切换 |
| 小门/入口 | 左上或上方 | 恢复状态使用 |
| 对话气泡 | 狗狗附近或状态栏内 | 不遮挡地图路线 |
| 狗狗状态栏 | 小世界底部内侧 | 点击狗狗后展开 |

### 层级

```text
地图底层：草地 / 地砖 / 石板路 / 院子边界
场景层：狗窝 / 小门 / 栅栏 / 花圃
道具层：球 / 书 / 毯子 / 脚印
狗狗层：当前品种 + 当前姿态
UI 层：对话气泡 / 狗狗状态栏 / 状态浮层
```

## 3. 狗狗位置与尺寸

目标：狗狗必须是小世界里的主角，但不是占满画面的宠物头像。

建议：

- 狗狗高度约占狗狗小世界高度的 14%-20%。
- 狗狗宽度约占狗狗小世界宽度的 9%-15%。
- 狗狗底部靠近场景下沿，留出脚下阴影。
- 狗狗不要贴边，左右至少留出 16px 安全边距。
- 对话气泡不遮挡地图里的狗狗和路线。
- 院子要像狗狗的生活空间，有足够活动空地。
- 第一眼更像一张小院地图，第二眼看到狗狗。
- 后期增加 2-3 只狗时，画面不能显得夸张拥挤。
- 地图里的狗狗可以较小，点击后由状态栏展示清楚的品种和独特外观。

## 4. 狗狗状态栏

目标：点击狗狗后，让用户看清狗狗样子、品种和状态，但不打断首页节奏。

位置建议：

- 放在狗狗小世界底部内侧。
- 像一个低矮的信息托盘。
- 默认收起，点击狗狗后展开。
- 不用居中大弹窗。
- 不跳转页面。

线框：

```text
┌──────────────────────────────┐
│       鸟瞰小院地图             │
│          [狗狗]               │
│                              │
│ ┌──────────────────────────┐ │
│ │ [狗狗像素形象] 阿土 · 田园犬 │ │
│ │ Lv.1  期待中   亲密度 12/20 │ │
│ └──────────────────────────┘ │
└──────────────────────────────┘
```

规则：

- 状态栏里的狗狗形象可以比地图上的狗狗更大。
- 状态栏显示当前点击的狗狗。
- 多狗版本中，点击不同狗切换状态栏内容。
- 状态栏可以承载狗狗短句，减少地图里气泡遮挡。

## 5. 状态位置

| 状态 | 狗狗位置 | 视觉感觉 |
| --- | --- | --- |
| 待完成 | 中下方，面向用户 | 等你开始 |
| 已完成 | 稍靠前，接近用户 | 开心回应 |
| 漏 1 天 | 靠近狗窝或小门 | 安静等待 |
| 恢复中 | 从小门旁回到中间 | 重新接上 |
| 长期中断 | 狗窝旁或小门旁 | 不责备，只等你 |

## 6. 道具区

目标：通过道具呼应用户目标，不改变狗狗品种。

道具区位置：

- 默认放在狗狗左侧或前侧。
- 不超过狗狗视觉权重。
- 保持同一位置，减少切换跳动。

道具组：

| 目标类型 | 道具 | 画面意图 |
| --- | --- | --- |
| 健身 | 球、运动垫、水碗 | 狗狗也准备动起来 |
| 学习 | 书、本子、台灯 | 狗狗安静陪读 |
| 作息 | 毯子、月亮灯、睡觉窝 | 狗狗准备收尾休息 |
| 恢复 | 小门、脚印、小垫子 | 狗狗在等你回来 |
| 已完成 | 亮一点的灯、满水碗 | 今天被接住 |

## 7. 对话气泡

目标：让狗狗看起来真的在回应用户。

位置：

- 优先在狗狗头部上方。
- 居左或居中都可以，但气泡尖角指向狗狗。
- 移动端宽度建议不超过小世界宽度的 76%。

样式方向：

- 温和、轻盈。
- 圆角但不要幼稚。
- 文案 1 句为主。
- 不使用大段解释。

线框：

```text
┌──────────────────────┐
│ 今天先动一下就好      │
└─────────┬────────────┘
          │
        [狗狗]
```

默认气泡文案：

| 状态 | 示例 |
| --- | --- |
| 待完成 | 今天先做一点就好。 |
| 点击狗狗 | 要开始了吗？我陪你。 |
| 已完成 | 今天的节奏接住了。 |
| 恢复 | 昨天乱了也没关系。 |
| 长期中断 | 我们把它调小一点吧。 |

## 8. 今日行动区

目标：让用户低成本完成今日任务。

结构：

```text
┌──────────────────────────────┐
│ 今日目标                      │
│ ┌──────────────────────────┐ │
│ │ 运动 20 分钟              │ │
│ │ 每天 · 今天未完成          │ │
│ └──────────────────────────┘ │
│                              │
│ [ 完成今天 ]                 │
│                              │
│ 本周节奏  ○ ● ● ○ ○ ○ ○      │
│ 亲密度 Lv.1  12/20   [进度]  │
└──────────────────────────────┘
```

层级：

1. 今日目标标题。
2. 目标卡片。
3. 主按钮。
4. 本周节奏。
5. 亲密度和进度入口。

规则：

- 主按钮必须足够大，单手可点。
- 今日目标卡片不做成重数据面板。
- 亲密度只是辅助，不压过打卡。
- 本周节奏展示 7 个点即可。

## 9. 恢复状态首页

适用：昨天未完成，今天进入恢复任务。

```text
┌──────────────────────────────┐
│ 节奏有点乱，没关系           │
│ 今天做小一点          [进度]  │
├──────────────────────────────┤
│  安静一点的小院子            │
│                              │
│ ┌────────────────────┐       │
│ │ 昨天乱了也没关系    │       │
│ └────────┬───────────┘       │
│          │                   │
│ [小门/脚印]    [狗窝]        │
│                              │
│       [ waiting 狗狗 ]        │
│                              │
├──────────────────────────────┤
│ 今日恢复任务                 │
│ 拉伸 3 分钟                   │
│ 比原目标更轻一点              │
│                              │
│ [ 做个小恢复 ]               │
│ [ 调小目标 ]                 │
│                              │
│ 亲密度不会下降                │

```

重点：

- 不出现失败和清零。
- 狗狗不生气。
- 恢复任务比原目标小。
- “调小目标”是次级动作。

## 11. 点击狗狗状态

适用：用户点击狗狗。

交互：

1. 狗狗短暂看向用户。
2. 气泡出现 3-5 秒。
3. 不跳页，不弹复杂菜单。

```text
┌──────────────────────────────┐
│                              │
│ ┌────────────────────┐       │
│ │ 我看到你来了        │       │
│ └────────┬───────────┘       │
│          │                   │
│        [ 看向用户的狗狗 ]     │
│                              │
└──────────────────────────────┘
```

文案根据当前状态选择：

| 当前状态 | 文案方向 |
| --- | --- |
| 待完成 | 陪你开始 |
| 已完成 | 肯定今天 |
| 恢复 | 回来就好 |
| 长期中断 | 目标调小 |

## 12. 完成打卡后的即时反馈

适用：用户点击“完成今天”后。

表现：

```text
┌──────────────────────────────┐
│  小院子更亮                  │
│                              │
│ ┌────────────────────┐       │
│ │ 做到了，真不赖      │       │
│ └────────┬───────────┘       │
│          │                   │
│        [ happy 狗狗 ]         │
│                              │
│     亲密度 +3                │
│     精力 +20                 │
└──────────────────────────────┘
```

轻动画建议：

- 狗狗轻微跳一下或摇尾巴。
- 亲密度 +3 向上浮动。
- 道具或小院子亮度轻微提升。

约束：

- 不做复杂结算页。
- 状态变化最多展示 2-3 条。
- 不出现负向扣分。

## 13. 移动端安全区与可读性

规则：

- 狗狗小世界的文字气泡不能贴近屏幕边缘。
- 主按钮底部应避开系统 Home Indicator。
- 今日行动区不要因为小屏幕挤压到按钮不可见。
- 目标名称过长时最多显示 2 行。
- 气泡文案过长时换行，不缩到难读。

## 15. 设计验收

- 首页第一眼是狗狗小世界，不是任务列表。
- 用户不读说明也能知道点哪里完成今天。
- 待完成、已完成、恢复状态能一眼区分。
- 狗狗品种没有任务绑定暗示。
- 小院子温暖、干净、有陪伴感。
- 恢复状态不责备、不惩罚、不破败。
- 对话气泡像狗狗说话，不像系统说明。

## 下一步

建议下一步进入两条线之一：

1. 产出首页视觉方向参考：温暖插画风的小院子、狗狗姿态、道具风格。
2. 开始 app 原型准备：先做一个移动端 Web MVP，跑通首页、领养、目标创建、打卡和反馈闭环。
